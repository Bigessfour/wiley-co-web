# Wiley Widget Functional Breakdown

## Scope

This review covers the currently wired Wiley Widget experience:

- App shell and routes in [Components](../Components)
- Client state and HTTP services in [State](../State) and [Services](../Services)
- Shared DTOs in [Contracts](../Contracts)
- Server endpoints in [WileyCoWeb.Api](../WileyCoWeb.Api)
- Back-end business logic in [src/WileyWidget.Services](../src/WileyWidget.Services) and [src/WileyWidget.Data](../src/WileyWidget.Data)

It focuses on active end-to-end behavior, not every dormant helper or legacy class under [src](../src).

## End-To-End Flows

### Workspace Load Flow

`Browser -> Home/Workspace route -> WorkspaceBootstrapService -> /api/workspace/snapshot -> WorkspaceSnapshotComposer -> Aurora repositories -> WorkspaceState -> WorkspacePersistenceService browser restore/save`

### Scenario Planning Flow

`Workspace shell -> scenario grid edits in WorkspaceState -> Save scenario / Apply scenario -> /api/workspace/scenarios -> BudgetSnapshots -> refreshed WorkspaceState`

### QuickBooks Import Flow

`QuickBooksImportPanel -> QuickBooksImportApiService -> /api/imports/quickbooks/preview or /commit -> QuickBooksImportService -> SourceFiles + ImportBatches + LedgerEntries`

### Jarvis Flow

`JarvisChatPanel -> WorkspaceKnowledgeApiService and WorkspaceAiApiService -> /api/workspace/knowledge and /api/ai/chat -> WorkspaceKnowledgeService + WorkspaceAiAssistantService -> conversation/recommendation persistence`

### Export Flow

`Workspace shell -> WorkspaceDocumentExportService -> BrowserDownloadService` for immediate client downloads, and `saved snapshot -> /api/workspace/snapshot/{id}/exports -> WorkspaceSnapshotExportArchiveService -> BudgetSnapshotArtifacts` for archived exports.

## E2E Proof Standard

For this pass, a function counts as E2E-proven only if the code shows all of the following:

1. A concrete entry trigger in the widget UI or a callable API route.
2. A request/DTO or state mutation path that carries the work.
3. An executor that actually performs the operation.
4. A terminal result that is observable in code: API response, database mutation, browser storage write, or downloaded file.

The tables below tighten the inventory to functions that can be traced all the way through. Use the functional-area sections later in this document for the supporting file map.

This is a code-trace proof standard, not an executed test certification. The flows are proven from the wired code paths and persistence/output endpoints, but they were not runtime-validated in this review.

## Proof Snapshot

+ Core workspace bootstrap, baseline persistence, rate snapshots, scenarios, workspace knowledge, QuickBooks preview and commit, Jarvis persistence and reset, and immediate export flows have strong proof in the current code and test inventory.
+ Client-contained routing, rate recomputation, customer filtering, browser persistence, and download flows are directly covered by passing browser or component tests.
+ The current AWS runtime shape for Aurora access, runtime secret injection, xAI proxying, and Amplify build-time secret delivery is verified for the deployed environment.
+ Hosted QuickBooks preview and assistant proof is now complete on the public Amplify site: job `43` deployed commit `8620cf72f569058a5c72619e9e341e25fb0b34f1`, and both hosted browser tests passed against `https://main.d2ellat1y3ljd9.amplifyapp.com` on 2026-04-16.
+ The shipped Customer Viewer now has browser proof for roster display, service filtering, and full create/edit/delete maintenance over `/api/utility-customers`, while keeping the shared customer summary/export state synchronized.
- Archived snapshot exports are proven only through the API today. No workspace-shell trigger ships in the current widget, and any dedicated shell action remains later product scope.
- Remaining observability work is post-release infrastructure only: the current App Runner X-Ray path is still unproven in production, and OpenTelemetry is the preferred successor path.

## Short Take

Wiley Widget is currently a focused utility rate-study shell with five major product capabilities:

1. Live workspace composition from Aurora-backed data.
2. Rate modeling through break-even, rate editing, snapshots, and scenarios.
3. Customer review plus exportable Excel/PDF deliverables.
4. QuickBooks ingestion with preview, duplicate blocking, and commit.
5. Jarvis decision support driven by analytics-backed knowledge and AI chat.
